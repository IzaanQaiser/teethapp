# Pearly — V1 User Flow

This document defines the happy-path flow for Pearly v1.
No edge cases are included.

---

## Primary Flow

1. User opens Pearly
2. Welcome screen
3. Trust & consent confirmation
4. How it works explanation
5. Baseline capture
6. Scan / analyzing animation
7. Results screen
8. Personalized plan preview
9. Paywall
   - User starts free trial OR closes paywall
10. Plan screen
    - Day 1 accessible
    - Day 2–3 collapsed
    - Day 4+ locked
11. Post-decision reassurance
12. Rescan framing

---

## Key Flow Rules

- Paywall is never shown before results
- Users always see Day 1 of the plan
- Locked content previews are visible
- No login or account creation at any point
- User can exit app at any step without data loss

---

## Retention Loop (Conceptual)

- Daily checklist completion
- Weekly rescan reminder
- Progress comparison reinforces habit
